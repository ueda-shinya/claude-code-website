## 0. コピー読解サマリー（デザイン判断の根拠）

| 観点 | 把握内容 | デザイン判断 |
|------|---------|------------|
| 文字量 | ヒーロー・こだわり・スタッフは長文あり | 読みやすい行長（max 65文字相当）と十分なline-height |
| 見出し階層 | h1×1（ヒーロー）/ h2×9（各セクション）/ h3×多数 | タイポグラフィスケールを3段階で明確に分ける |
| CTA数 | メインCTA×2（ヒーロー）+Instagram誘導（ギャラリー）+問い合わせフォーム+Googleマップ | CTAの視覚ヒエラルキーを--primary/--ghostで管理 |
| タブUI | メニューセクションにコーヒー/スイーツ タブ切り替え | JS制御のタブ。非アクティブはopacity制御でlayoutプロパティ禁止 |
| アコーディオン | FAQ 8問 | max-heightを使わずopacity+transformで対応 |
| カード数 | メニュー6品×2タブ、お客様の声4件、スタッフ3名 | グリッドレイアウト（SP:1列/TAB:2列/PC:3列） |
| フォームフィールド | 5フィールド＋Honeypot | モバイル最適化（フォントサイズ16px以上で自動ズーム防止） |
| ギャラリー | 写真6枚 | SP:2列/TAB:3列/PC:3列 正方形アスペクト比 |

### Section 4: メニュー（p-menu）

**目的**: タブ切り替えで12商品を視覚的訴求

| 項目 | SP | TAB | PC |
|------|----|----|-----|
| タブUI | 横並び2タブ（各50%幅） | 同左 | 同左 |
| カードグリッド | 1列（縦積み） | 2列（gap 1.5rem） | 3列（gap 2rem） |
| カード高さ | 自動（コンテンツに依存） | 同左 | 同左 |
| 画像エリア | 画像 aspect-ratio: 4/3 | 同左 | 同左 |
| 背景 | `--color-bg-primary` | 同左 | 同左 |

**タブの切り替え仕様:**
- 非アクティブパネルは `opacity: 0` + `transform: translateY(8px)` でフェードアウト
- アクティブパネルは `opacity: 1` + `transform: translateY(0)` でフェードイン
- display:none / display:block の切り替えは `is-active` クラスで管理
- **max-height・width等のlayoutプロパティによるアニメーション禁止**

**HTMLコンテンツ構造:**
```
p-menu
  ├── p-menu__header（h2 + リードコピー）
  ├── p-menu__tabs（role="tablist"）
  │     ├── button.c-tab js-tab（role="tab" aria-selected data-target="coffee"）コーヒー
  │     └── button.c-tab js-tab（role="tab" aria-selected data-target="sweets"）スイーツ
  └── p-menu__panels
        ├── div.p-menu__panel#coffee（role="tabpanel"）
        │     └── .p-menu__grid
        │           └── .c-card.c-card--menu × 6
        │                 ├── .c-card__image（aspect-ratio:4/3）
        │                 ├── .c-card__body
        │                 │     ├── h3.c-card__name（商品名）
        │                 │     ├── .c-card__price（価格）
        │                 │     └── p.c-card__desc（説明文）
```

- 開閉は `is-open` クラスのトグルで制御
- 答えパネル: `opacity: 0 + transform: translateY(-8px)` → `opacity: 1 + transform: translateY(0)`
- **max-heightアニメーション禁止** → 代替: JSでpanel要素に `is-open` を付与し、CSSで `display:grid; grid-template-rows: 0fr → 1fr` を transition（ただしgrid-template-rowsのtransitionはFirefox旧版非対応のため、 `opacity + transform` のみで対応）
- 実装: `is-open` 時に答えを `visibility:visible; opacity:1; transform:translateY(0)` / 非open時は `visibility:hidden; opacity:0; transform:translateY(-8px)` + `position:absolute` でレイアウトに影響させない
- 矢印アイコン: `transform: rotate(0deg)` → `transform: rotate(180deg)`

## 7. 信頼設計（視覚的見せ方）

### 7-1. 実績バナー（p-stats）

- ブランドブラウン背景で「ページの中断」をつくり、数字が自然に目に入る
- 数字はPlayfair Display 4rem でビジュアルインパクト最大化
- 4要素を均等グリッドで並べ、視覚的対称性で信頼感を強調
- 数字のカウントアップアニメーション（`transform: scale(1.05)` + `opacity` でスプリング感を演出。countUpはJSで数値インクリメント、DOMはtextContentで更新）

### 7-2. お客様の声（p-testimonials）

- 大きな引用符（`"` / `"` ）をbefore疑似要素で配置し、「肉声」感を演出
- 名前・年齢・職業を必ず表示し、リアルな人物像を明示
- カードの枚数（4件）をセクション見出し直下にリード文として示し「多数の声がある」ことを暗示
- 2列グリッドで「量感」を視覚的に示す

### 7-3. スタッフ紹介（p-staff）

- 写真は正円（border-radius:50%）で「顔」を前面に出す
- ひとこと引用にblockquoteタグ＋左ボーダー（4px solid --color-accent-green）で視覚的に際立たせる
- 役職バッジ（c-badge: 背景 `--color-accent-green-bg` テキスト `--color-accent-green-dark`）で専門性を可視化

### 7-4. 証拠の配置順序（信頼の積み重ね）

```
実績バナー（数字）→ 定量的信頼
     ↓
お客様の声（テキスト）→ 定性的・感情的信頼
     ↓
スタッフ紹介（顔・名前）→ 人的信頼（実在感）
```

## 8. CTA/フォームのUX仕様

### 8-1. CTAのヒエラルキー

| 優先度 | スタイル | 用途 |
|--------|---------|------|
| 第1（メインCTA） | c-button--primary | ヒーローCTA1（来店のご案内を見る） |
| 第2（サブCTA） | c-button--ghost | ヒーローCTA2（Instagramをフォローする）、各誘導ボタン |
| 第3（テキストリンク） | c-text-link（underline付き） | FAQ内リンク、フッターリンク |

### 8-2. フォームUX仕様

**入力負荷の軽減:**
- フィールドは縦1列配置（並列は入力ミスを増やす）
- `autocomplete` 属性を全フィールドに設定（`name="name" autocomplete="name"` 等）
- プレースホルダーはラベルの代替に使わない（ラベルは常に表示）
- 電話番号は任意（ハードル最小化）
- セレクト選択肢は5個以内

**エラー表示UX:**
- リアルタイムバリデーション: `blur` イベントでフィールドごとにチェック
- エラーは入力欄の直下に `c-form__error` で表示（赤テキスト + アイコン）
- エラーがある状態でSubmitを叩いた場合、最初のエラーフィールドへスクロール + `focus()`
- エラーメッセージはARIA: `role="alert"` + `aria-live="polite"`

**スマホ最適化:**
- 全inputのfont-size: 1rem（16px）以上 → iOS SafariのピンチZoom防止
- セレクトボックスは `font-size: 1rem`
- タップターゲット最小 44px × 44px（送信ボタンは56px以上）
- `inputmode` 属性: メールは `inputmode="email"`、電話は `inputmode="tel"`

**送信フロー:**
1. Submit → ボタンを `disabled` + テキスト「送信しています...」に変更
2. 成功 → フォームを `opacity:0 + translateY(8px)` でフェードアウト → サクセスメッセージを `opacity:1 + translateY(0)` でフェードイン
3. エラー → エラーブロックを表示 + ボタンを再度有効化

## 10. アクセシビリティ要件

### 10-1. フォーカス管理

```css
/* フォーカスリングは全要素に明示 */
:focus-visible {
  outline: 3px solid var(--color-border-focus);
  outline-offset: 2px;
  border-radius: 2px;
}

/* ハンバーガーメニューopen時: bodyのスクロール禁止 */
body.is-nav-open {
  overflow: hidden;
}
```

### 10-2. ARIA 必須属性一覧

| コンポーネント | 必須属性 |
|--------------|---------|
| ハンバーガーボタン | `aria-label="メニューを開く"` / `aria-expanded` / `aria-controls` |
| ナビゲーション | `<nav aria-label="メインナビゲーション">` |
| タブリスト | `role="tablist"` |
| タブボタン | `role="tab"` / `aria-selected` / `aria-controls` |
| タブパネル | `role="tabpanel"` / `aria-labelledby` |
| アコーディオントリガー | `aria-expanded` / `aria-controls` |
| アコーディオンパネル | `id` / `role="region"` / `aria-labelledby` |

| 全画像 | `alt`（装飾画像は `alt=""`） |
| フォームラベル | `for` でinputの `id` と紐付け |
| skip nav | `<a href="#main" class="u-visually-hidden">メインコンテンツへスキップ</a>` |
