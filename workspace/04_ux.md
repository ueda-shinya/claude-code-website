### 6-1. スクロールリビール（.js-reveal → .is-visible）

**対象要素:** 全セクションの見出し・カード・テキストブロック

**CSS定義:**
```css
/* 初期状態（非表示） */
.js-reveal {
  opacity: 0;
  transform: translateY(24px);
  transition:
    opacity 0.65s ease,
    transform 0.65s ease;
}

/* JS付与後（可視状態） */
.js-reveal.is-visible {
  opacity: 1;
  transform: translateY(0);
}

/* 遅延バリエーション（data属性で制御） */
.js-reveal[data-delay="100"] { transition-delay: 0.1s; }
.js-reveal[data-delay="200"] { transition-delay: 0.2s; }
.js-reveal[data-delay="300"] { transition-delay: 0.3s; }
.js-reveal[data-delay="400"] { transition-delay: 0.4s; }
.js-reveal[data-delay="500"] { transition-delay: 0.5s; }
```

**遅延設定の方針:**
- セクション見出し: delay="0"
- カードグリッド内: delay="100" × カード番号（最大500msまで）
- 一度可視化されたら遅延は不要（Intersection Observer: once: true）

**JS側の制御:**
```javascript
// IntersectionObserver で .js-reveal を監視
// rootMargin: '0px 0px -60px 0px'
// threshold: 0.1
// 交差したら .is-visible を付与（once: true）
```

### 6-2. FAQアコーディオンアニメーション

**方針:** `max-height` を `0 → scrollHeight` でトランジション（layoutプロパティ扱いだがcontentのみのため許容）
あるいは: `transform: scaleY` と `transform-origin: top` の組み合わせ

**推奨実装（CSSのみ版）:**
```css
.c-accordion__panel {
  max-height: 0;
  overflow: hidden;
  transition: max-height 0.35s ease;
}

.c-accordion.is-open .c-accordion__panel {
  max-height: 500px; /* 十分な高さ（コンテンツが500px以内の想定） */
}
```

**JS側:** `hidden`属性を`is-open`クラスと同時に切り替える
`aria-expanded` を `true / false` に切り替える

## 7. 信頼設計（実績・証言・数字の視覚的見せ方）

---

### 7-1. 実績バナーのデザイン

- **背景色**: var(--color-primary)（ウォームブラウン）— ページ内で唯一の濃い色帯としてアイキャッチ
- **数字フォント**: Playfair Display（欧文 serif）、font-size: SP 2.4rem / PC 4rem
- **数字色**: var(--color-text-on-dark)（オフホワイト）
- **ラベル色**: rgba(250, 247, 242, 0.75)（数字より控えめに）
- **区切り線**: rgba(250, 247, 242, 0.30)の縦線
- **ゼロからカウントアップするアニメーション**: JSで実装（Intersection Observer + requestAnimationFrame）
  - 数字のみ対象（「毎日焼き上げ」のように文字列のアイテムはカウントアップ対象外）
  - duration: 1500ms、easing: ease-out

---

### 7-2. お客様の声の引用符・強調方法

- **引用符装飾**: CSS `::before` 擬似要素で `"` を左上に大きく配置（5rem、color-accent-light、opacity: 0.5）
- **本文フォントサイズ**: SP: 0.8125rem、PC: 0.875rem（読みやすさと情報密度のバランス）
- **行間**: 1.85（ゆったり読める）
- **氏名**: font-weight: 700、color-text-primary（信頼感を与える明確な表示）
- **属性**: font-size: 0.8125rem、color-text-muted（具体性を持たせながら控えめに）
- **カード背景**: var(--color-bg-card)（白）で、セクション背景var(--color-bg-section-alt)から浮かせる

---

### 7-3. スタッフ紹介の顔写真処理

- **border-radius**: 50%（完全な円形）
- **サイズ**: SP: 120×120px / TAB: 140×140px / PC: 160×160px
- **border**: 3px solid var(--color-accent-light)（セージグリーン薄め）— 人の温かみを演出
- **box-shadow**: var(--shadow-sm)（わずかな立体感）
- **object-fit**: cover / **object-position**: top center（顔が切れないよう上基準）

## 8. CTA / フォームのUX

---

### 8-1. メインCTAボタンの配置とデザイン（ヒーロー内）

- **配置**: ヒーローコンテンツの最下部、サブコピー3文の直下
- **SP**: CTAボタン2つを縦積み（flex-direction: column）、幅: min(100%, 320px)
- **TAB/PC**: CTAボタン2つを横並び（flex-direction: row）
- **ボタン順序**: Instagramをフォローする（primary/instagram）→ アクセスを確認する（secondary/outline-white）
- **サイズ**: min-height: 52px（通常の48pxより少し大きく）、padding: 14px 32px
- **Instagram CTAは `.c-button--instagram` ではなくヒーローでは目立つよう `.c-button--primary` 相当のブラウンに統一する場合もあるが、マーケティング戦略書のメインCTAをInstagramと定義しているため `.c-button--instagram`（ピンク）を使用してInstagramと視覚的に紐付ける**

---

### 8-2. ギャラリーInstagram CTAのデザイン

- **エリア**: ギャラリーグリッド下部、中央寄せ
- **リード文**: font-size: var(--text-base)、color: var(--color-text-secondary)、margin-bottom: 24px
- **ボタン**: `c-button c-button--instagram`
- **SP**: 幅100%（max-width: 320px, margin: 0 auto）
- **PC**: 幅auto（インライン）

---

### 8-3. フォームの入力フィールドUX

**タップターゲット:**
- 全input/select/textarea: min-height: 48px（タップターゲット最小44px超を確保）
- チェックボックスの実際のclick/tap可能エリア: label全体（20×20pxのcheckbox+テキスト行）

**エラー表示位置:**
- 各フィールドの直下に `.c-form__error`（role="alert"）を配置
- エラーテキストは aria-live="polite" で動的に挿入（スクリーンリーダー対応）
- エラー発生時: フィールドに `.is-error` クラスを付与（ボーダー赤）
- エラーが解消されたとき: `.is-error` 除去、aria-live で「」（空文字）を挿入

**スクロールポジション:**
- バリデーションエラーの場合、最初のエラーフィールドに `scrollIntoView({ behavior: 'smooth', block: 'center' })` でフォーカス移動

**ラベル設計:**
- 必須フィールド: ラベル末尾に `*`（`.c-form__required`）を赤色で表示
- 任意フィールド:「（任意）」のテキストをラベル末尾に付加
